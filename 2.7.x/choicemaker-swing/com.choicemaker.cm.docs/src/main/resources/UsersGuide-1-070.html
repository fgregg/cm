1 Approximate string matching

ChoiceMaker provides several classes for approximate string matching in plugins com.choicemaker.cm.matching.x, where x stands for a locale or `gen' for generic. Examples that illustrate the tradeoffs of the different methods can be found in Section 1.9. ModelMaker contains a utility to explore these functions (Chapter II.12.2).

1.1 Soundex

Soundex ((Knuth and Donald 1998)) is a standard phoneticization algorithm. It is a very simple algorithm that works especially well for English. Many databases have a built-in Soundex function that can be used for creating derived fields for blocking.

ChoiceMaker's implementation of Soundex is provided by the class com.choicemaker.cm.matching.en.Soundex. The class provides two static public methods:

Method

Semantics

String soundex(String s, int numDigits)

Returns the Soundex of s to numDigits accuracy. numDigits must be non-negative.

String soundex(String s)

Same as soundex(s, 3)

1.2 NYSIIS

NYSIIS, the New York State Information Interchange System, (Newcombe 1988) is a phoneticization algorithm similar to Soundex. It also works especially well for English. NYSIIS accounts for more phonetic spelling errors than Soundex and at the same time maps fewer strings with different pronunciations to the same code.

Unlike Soundex, NYSIIS is slightly modified by most vendors. Hence, NYSIIS codes are not interchangeable.

ChoiceMaker's implementation of NYSIIS is provided by the class com.choicemaker.cm.matching.en.Nysiis. The class provides a single static public method:

Method

Semantics

String nysiis(String s)

Returns the NYSIIS code of s.

1.3 Metaphone

Metaphone (Philips 1990) is another phoneticization algorithm. It works especially well for English. ChoiceMaker's implementation is provided by the class com.choicemaker.cm.matching.en.Metaphone. The class provides a single static public method:

Method

Semantics

String metaphone(String s)

Returns the Metaphone code of s.

1.4 Double metaphone

Double metaphone (Philips 2000) is an improved version of metaphone. It works especially well for English. For certain strings that may have more than one pronunciation, double metaphone returns two codes, a primary and an alternate.

ChoiceMaker's implementation is provided by the class com.choicemaker.cm.matching.en.DoubleMetaphone. It provides a public static method doubleMetaphone and several instance methods:

Method

Semantics

DoubleMetaphone doubleMetaphone(String s)

Creates a DoubleMetaphone instance that holds the primary and, if it exists, alternate code.

boolean equals(Object o)

Returns true if the primary or non-null alternate codes are equal. This includes equality between the primary code of one object and the non-null alternate of the other.

boolean primaryEquals(DoubleMetaphone o)

Returns true if the primary codes agree.

String getPrimary()

Returns the primary code.

String getAlternate()

Returns the alternate code.

1.5 Jaro-Winkler

The Jaro-Winkler algorithm (Jaro 1989) (Porter and Winkler ) is a string comparator developed by two researchers at the US Census Bureau. It computes a similarity score between two strings in the range 0 to 1. It works especially well for English. Differences at the beginning are weighed more strongly than differences towards the end of the strings.

ChoiceMaker's implementation is provided by the class com.choicemaker.cm.matching.en.Jaro. The class provides two public static methods:

Method

Semantics

float jaro(String a, String b,

boolean higherScoreForLongStrings,

boolean convertToUpper,

boolean checkForNumbers)

Computes the Jaro-Winkler similarity between a and b.

higherScoreForLongStrings: Increase the probability of a match when the number of matched characters is large. This option allows for a little more tolerance when the strings are large. It is not an appropriate test when comparing fixed length fields such as phone and social security numbers.

convertToUpper: All lower case characters are converted to upper case prior to the comparison.

checkForNumbers: Check for numbers in data; if number is present in either string and the strings do not match, then 0.0 is returned, else the calculated weight is returned.

float jaro(String a, String b)

Same as jaro(a, b, true, true, false)

1.6 Edit distance

Edit distance computes the minimum number of edits that have to be made to one string to transform it into the other. Deletion, insertion, and replacement of a letter are the possible edit steps. Edit distance can be used to detect typos rather than phonetic misspellings. The function is equally well suited for most languages with a small alphabet. E.g., the edit distance between `Test' and `Fest' is 1. The edit distance between `Test' and `Feste' is 2 because we have to perform one deletion and one replacement to transform `Feste' into `Test'.

ChoiceMaker's implementation is provided by the class com.choicemaker.cm.matching.gen.EditDistance. It has two public static methods:

Method

Semantics

int editDistance(String a, String b,

int maxDistance)

Computes the edit distance between a and b. If the result would be greater than maxDistance, then maxDistance + 1 is returned. The rationale for the maxDistance parameter is to improve performance by stopping the computation if the result would be bigger than what we are interested in.

int editDistance(String a, String b)

Same as editDistance(a, b, Integer.MAX_VALUE -1)

1.7 Longest common substring

The longest common substring algorithm (Friedman and Sideli 1992) provides two functions. First, it computes the length of the longest common substring between two strings. E.g., the longest common substring of "TEST" and "FEST" is "EST", which has a length of 3.

The second function is a similarity score. The function repeatedly removes the longest common substring from the two strings to be compared. The similarity score is defined by the sum of the lengths of the removed substrings divided by some measure. The latter can be the average, minimum, or maximum length of the original two strings.

The longest common substring similarity score can be used, for example, when values are reversed. E.g., the comparison of "JIM SMITH" with "SMITH JIM" returns a perfect score of 1. The function is equally well suited for most languages with a small alphabet.

ChoiceMaker's implementation is provided by com.choicemaker.cm.matching.gen.LongestCommonSubstring. It provides three public static methods:

Method

Semantics

int longestCommonSubstring(String a,

String b)

Returns the length of the longest common substring.

float similarity(String a, String b,

int denominator, int minLength,

int maxRepetition)

Computes the similarity between a and b.

denominator defines what value is used as the denominator for computing the similarity. It can be one of the following:

AVERAGE: (a.length() + b.length()) / 2f

SHORTEST: min(a.length(), b.length())

LONGEST: max(a.length(), b.length())

minLength determines the minimum length of a substring that is removed.

maxRepetition: The maximum number of removal iterations.

float similarity(String a, String b)

Same as similarity(a, b, SHORTEST, 3, 3)

1.8 Longest common subsequence

The longest common subsequence provides two functions. The first returns whether one string is a subsequence of another. That is, whether all the characters of the shorter string appear in the same order in the longer string. For example, "BELLEVILLE COMM CLG" is a subsequence of "BELLEVILLE COMMUNITY COLLEGE", "CMT" is a subsequence of "CHOICEMAKER TECHNOLOGIES", "ADAM" is a subsequence of "ADAM JOHN".

This function comes in two flavors: Versions that require the first character to be the same and ones that don't. E.g., "COMM CLG" is not a subsequence of "BELLEVILLE COMMUNITY COLLEGE" for the first, but it is for the second. Additionally, there is a parameter for the minimum length to avoid a match between "C" and "CHOICEMAKER TECHNOLOGIES".

The second function returns the length of the longest common subsequence and the third returns the actual subsequence.

ChoiceMaker's implementation is provided by com.choicemaker.cm.matching.gen.LongestCommonSubsequence. It provides three public static methods:

Method

Semantics

boolean isLcsAbbrev(String w1, String w2)

Returns whether one of the two strings of length at least 2 is a subsequence of the other and they start with the same characters. More specifically, returns true iff:

  1. the minimum of the lengths of w1 and w2 is at least 2, and
  2. the longest common subsequence length of w1 and w2 is equal to the minimum of the lengths of w1 and w2, and
  3. w1 and w2 begin with the same character.

boolean isLcsAbbrev(String w1, String w2, int minLength)

Returns whether one of the two strings of length at least minLength is a subsequence of the other and they start with the same characters. More specifically, returns true iff:

  1. the minimum of the lengths of w1 and w2 is at least minLength, and
  2. the longest common subsequence length of w1 and w2 is equal to the minimum of the lengths of w1 and w2, and
  3. w1 and w2 begin with the same character.

isLcsAbbrevAnyStart(String w1, String w2, int minLength)

Returns whether one of the two strings of length at least minLength is a subsequence of the other and they start with the same characters. More specifically, returns true iff:

  1. the minimum of the lengths of w1 and w2 is at least minLength, and
  2. the longest common subsequence length of w1 and w2 is equal to the minimum of the lengths of w1 and w2.

int lcsLength(String x, String y)

Returns the length of the longest common subsequence between x and y.
